How this works
1) RetrievalBM25 + dense
We retrieve candidates using lexical BM25 and semantic dense embeddings. Then we merge them into a hybrid candidate list.
2) LTR Re-rankingLambdaRank
We compute ranking features (BM25 score, dense score, metadata signals) and apply a LightGBM LambdaRank model. This improves offline nDCG@10 and produces a more “Netflix-like” ordering.
3) Grounded + Agentic RAGsafe explanations
The LLM answers using only retrieved metadata snippets (grounding) and must cite sources. If citations are missing or evidence is weak, an agent retries with more candidates/context.
Reality check: MovieLens has limited language/region metadata. Language filtering in this demo is best-effort.
What Netflix gets from this
- Search relevance stack with offline eval + regression gates
- LTR training + inference pipeline (separate artifact build steps)
- Product-like discovery UI that demonstrates ranking lift + explainability
- Agentic RAG pattern to reduce hallucinations and improve “why this title” explanations
It’s a strong prototype for relevance infra — not a full Netflix MLP platform replacement.
