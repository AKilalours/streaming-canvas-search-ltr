Engineering Decisions

Trade-offs Called Out

Every decision has a cost. Every cost was measured. No defaults were left unexamined.

⚡ Latency vs Quality
α=0.2 (BM25-dominant). Dense adds 40ms but improves long-tail recall. Fine-tuning added 30 min of training but +8.3% LTR nDCG permanently.
Decision: Accept all latency costs. Cache absorbs 80%+ at 2.67ms. Quality gains are permanent.
💰 Freshness vs Cost
GPT-4o-mini explanations cached per (doc_id, profile, language). Don't refresh with new reviews. Film descriptions are stable.
Decision: Session cache. $0.0008 → $0.00 repeat views. Staleness acceptable for film metadata.
🎲 Exploration vs Exploitation
ε=0.15 reserves 15% of feed slots for outside genre history. Sacrifices short-term CTR for long-term discovery. IPW corrects for bias in off-policy eval.
Decision: Accept CTR drop. Doubly-robust IPW ensures exploration doesn't corrupt measurement.
📐 Model Size vs Performance
e5-base beats e5-large on this corpus (0.55 vs lower). Fine-tuned base beats un-tuned large. Larger is not always better on short-title queries.
Decision: Fine-tune the base model. 4× less memory, faster inference, better measured results.

Postmortem

What Broke and How I Fixed It

Real failures, real fixes. Every bug made the system more robust.

01
Dense Model Below BM25 Baseline BROKEFIXED +52% then +18.4%
What broke: all-MiniLM-L6-v2 scored 0.30 — worse than BM25 at 0.61.
Fix 1: Switched to e5-base-v2 (768-dim, instruction-tuned). Added "query:"/"passage:" prefixes. 0.30 → 0.4640.
Fix 2: Fine-tuned on MovieLens domain data. 0.4640 → 0.5496. LTR 0.8589 → 0.9300.
Lesson: Always benchmark multiple models. Fine-tuning on domain data compounds through every downstream stage.
02
BEIR Below Published Reference BROKEFIXED +19%
What broke: 0.2712 — below Thakur et al. 2021 reference of 0.325.
Fix: Porter stemming on medical vocabulary. "antineoplastic agents" → matches "antineoplastic". 0.2712 → 0.3236.
Lesson: Domain-specific tokenization is critical. Never assume default tokenizers handle medical text.
03
WebSocket Code 1006 Disconnect BROKEFIXED
What broke: All WebSocket connections dropped immediately. Code 1006 abnormal close.
Fix: "import asyncio as _asyncio" but endpoint called asyncio.wait_for directly — NameError. Changed all asyncio.* → _asyncio.*. Added 25-second keepalive ping.
Lesson: Import aliases propagate silently. Always load-test WebSocket under real connection scenarios.
04
GPT-4o Vision Silent Failure BROKEFIXED
What broke: /vlm/describe_poster returned empty string silently. No error logged.
Fix: gpt-4-turbo doesn't accept base64. TMDB URLs blocked by OpenAI. Switched to gpt-4o + download image → base64 encode → data:image/jpeg;base64,...
Lesson: Test vision endpoints with actual images in exact API format. Model capabilities are poorly documented.
05
A/B Test Underpowered — Honest Reporting HONEST CALL
What happened: p=0.065, lift=+6.3%, MDE=0.031. Lift below MDE. Not significant at n=200.
Decision: Did not ship. Reported underpowered result honestly. Real A/B needs 10K+ users.
Lesson: Honest underpowered results are more credible than p-hacked significance. The IPW machinery is production-correct regardless.
Interview Prep

System Design Questions — Answered

Time yourself. These are the exact questions you'll get from senior ML engineers.

Design a RAG for 1M PDFs. Latency <1.5s. Where do caching and rerankers live?
Clarify first: query latency or indexing latency? Avg or p99?

Stage 1 — retrieval: BM25 + fine-tuned e5 → 500 candidates in ~200ms (same pattern as StreamLens).
Stage 2 — reranker: LightGBM cross-encoder → top-20 in ~300ms.
Stage 3 — LLM: GPT-4o-mini with top-5 chunks → answer in ~700ms.
Total: ~1.2s. Cache on (query_hash, chunk_ids) in Redis → p50=2.67ms repeats.

Where latency hides: tokenization (50ms), FAISS search (100ms), reranker (300ms), LLM (700ms). Cache everything you can before the LLM.
Deploy LLM assistant with small→big routing, cost guardrails, fail-open paths.
Routing logic: query complexity score → if <0.4: Llama3 local ($0), if >0.4: GPT-4o-mini ($0.0008), if >0.8: GPT-4o ($0.008).

Cost guardrails: Redis counter per user per day. Hard cap $0.10/user/day. Throttle to local model after cap.

Fail-open: GPT-4o-mini down → Ollama Llama3 → template fallback. Never return empty. Always return something.

StreamLens implementation: exactly this chain, measured at $0.0008/request with 84% headroom under budget.
Make it resilient to data drift. Describe eval gates, rollbacks, shadow tests.
Eval gates: 9 quality gates must pass before model promotion. nDCG@10 > 0.80, BEIR > ref, p99 < 200ms. Any failure → block promotion + alert.

Shadow mode: new model runs in parallel, logs scores without serving. If shadow beats prod by >2% for 24h → A/B test → gradual rollout.

Rollback: Metaflow artifact versioning. Previous model artifact always retained. railway rollback in 30 seconds.

Drift detected: 24.6% gap in pre-2010 content identified via temporal analysis. Quantified and roadmapped — not ignored.
Where does latency hide in this system? How do you measure cost per request?
Latency breakdown (cold):
BM25 retrieval: ~5ms
FAISS ANN search: ~15ms
Feature computation: ~10ms
LightGBM rerank: <1ms (sub-ms!)
Redis overhead: ~2ms
Network + FastAPI: ~10ms
Total: ~142ms p99 cold

Cost per request: GPT-4o-mini = $0.0008. Cached = $0.00. Ollama = $0.00. Weighted average at 80% cache hit: ~$0.00016/request effective cost.
